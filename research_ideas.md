# Идеи для расширения исследования

Ниже перечислены идеи, которые можно добавить в `main_2.py` без радикальной перестройки текущей архитектуры. Большинство из них можно оформить как отдельные функции, которые запускаются после обучения финальной модели или сразу после загрузки данных.

## 1. ROC Curve и ROC-AUC

Добавить расчет `roc_auc_score` и построение ROC-кривой для моделей, у которых есть `predict_proba`.

Польза: стандартный академический способ оценить бинарный классификатор независимо от фиксированного порога `0.5`.

## 2. Precision-Recall Curve и PR-AUC

Добавить `precision_recall_curve` и `average_precision_score`.

Польза: особенно полезно, если классы несбалансированы или важнее качество по положительному классу.

## 3. Threshold Analysis

Проверить качество модели при разных порогах классификации: `0.1`, `0.2`, ..., `0.9`.

Польза: показывает компромисс между precision и recall.

## 4. Best Threshold Selection

Найти порог, который максимизирует `F1`, `balanced_accuracy` или другую целевую метрику.

Польза: можно сравнить стандартный threshold `0.5` с оптимизированным threshold.

## 5. Calibration Curve

Построить calibration curve для вероятностных моделей.

Польза: показывает, насколько вероятности модели соответствуют реальным частотам класса.

## 6. Brier Score

Добавить `brier_score_loss` для оценки качества вероятностей.

Польза: дополняет ROC-AUC, потому что оценивает не только ранжирование, но и качество вероятностных оценок.

## 7. Confidence Analysis

Посчитать среднюю уверенность модели отдельно для правильных и ошибочных предсказаний.

Польза: показывает, ошибается ли модель уверенно или в основном на пограничных случаях.

## 8. Top Confident Errors

Сохранять CSV с самыми уверенными ошибками модели.

Польза: сильный блок для error analysis, потому что показывает наиболее опасные ошибки.

## 9. False Positive / False Negative Profiles

Сравнить средние значения признаков для `false_positive`, `false_negative` и `correct`.

Польза: помогает понять, какие группы объектов чаще приводят к разным типам ошибок.

## 10. Error Rate by Feature Bins

Для числовых признаков создать интервалы и посчитать error rate в каждом интервале.

Польза: можно увидеть, например, хуже ли модель работает на очень длинных перелетах или при больших задержках.

## 11. Permutation Importance

Добавить `sklearn.inspection.permutation_importance`.

Польза: показывает важность признаков именно для финальной модели, а не только статистическую связь признака с target.

## 12. Feature Importance для Tree-based Models

Для Random Forest и XGBoost сохранять `feature_importances_`.

Польза: простой способ интерпретировать деревья и ансамбли.

## 13. Сравнение SelectKBest и Model Importance

Сравнить ранжирование признаков из `SelectKBest` с permutation importance или tree importance.

Польза: исследовательски интересно, потому что разные методы важности могут давать разные выводы.

## 14. Correlation Matrix для Числовых Признаков

Построить и сохранить корреляционную матрицу числовых признаков.

Польза: помогает увидеть мультиколлинеарность и связанные группы признаков.

## 15. Target Rate by Category

Для категориальных признаков посчитать долю `satisfied` внутри каждой категории.

Польза: показывает, какие категории связаны с большей или меньшей удовлетворенностью.

## 16. Target Rate by Numeric Bins

Для числовых признаков разбить значения на интервалы и посчитать долю `satisfied`.

Польза: дает интерпретируемый анализ зависимости target от признаков.

## 17. Class Balance Report

Сохранять таблицу с количеством и долей классов в train и test.

Польза: фиксирует, насколько сбалансирована задача и одинаково ли распределен target в train/test.

## 18. Train-Test Target Drift

Сравнить распределение target в train и test.

Польза: дополняет drift analysis признаков и проверяет, не отличается ли доля satisfied в test.

## 19. Missing Values Report

Сохранять отдельную таблицу по пропускам в train/test для всех признаков.

Польза: удобно для раздела предварительного анализа данных.

## 20. Duplicate Rows Check

Проверить количество дубликатов в train и test.

Польза: показывает качество данных и помогает обнаружить возможные проблемы с датасетом.

## 21. Outlier Summary

Для числовых признаков посчитать долю выбросов по IQR-правилу.

Польза: помогает описать особенности распределений и потенциальное влияние выбросов.

## 22. Model Ranking Table

Создать итоговую таблицу сравнения моделей по `F1`, `accuracy`, `balanced_accuracy`, fit time и predict time.

Польза: удобно для финального вывода и выбора лучшей модели.

## 23. Quality vs Speed Plot

Построить график: по оси X время предсказания, по оси Y `F1-weighted`.

Польза: показывает trade-off между качеством и скоростью.

## 24. Robustness Summary by Model

Если robustness запускался для нескольких моделей, собрать общий CSV с максимальным падением F1.

Польза: можно сравнить не только качество моделей, но и устойчивость.

## 25. Fairness Summary by Model

Собрать общий CSV с максимальными `f1_gap`, `error_rate_gap`, `recall_positive_gap` по каждой модели.

Польза: помогает сравнить модели по fairness-критериям.

## 26. Statistical Comparison of Models

Добавить McNemar test для сравнения двух моделей на одном test set.

Польза: можно проверить, является ли различие в ошибках статистически значимым.

## 27. Bootstrap Comparison Between Models

Использовать bootstrap не только для одной модели, но и для разницы метрик между двумя моделями.

Польза: позволяет построить доверительный интервал для разницы качества.

## 28. Cross-Validation Stability Report

Сохранять не только средний CV score, но и разброс score по folds.

Польза: показывает, насколько стабильна модель на разных разбиениях.

## 29. Learning Curve Interpretation Table

По learning curve автоматически сохранять разницу между train score и validation score.

Польза: помогает формально обсуждать underfitting/overfitting.

## 30. Reproducibility Report

Сохранять в CSV или TXT основные параметры эксперимента: `RANDOM_STATE`, модель, best params, размеры train/test, дата запуска.

Польза: улучшает воспроизводимость экспериментов и делает работу более аккуратной.
